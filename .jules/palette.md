## 2025-01-20 - Ensure Keyboard Accessibility with focus-visible Styles
**Learning:** Adding custom UI elements, like buttons (`<button>`) or custom list items acting as interactive controls (e.g., `.entry-item.active`), can inadvertently omit focus styles if they do not inherit global styles or if custom styles overwrite default outlines. This creates an accessibility issue for keyboard users navigating through interactive components, as they may lose track of the currently focused element. Combining existing visual effects like inset shadows with focus ring styles allows visual consistency without overwriting important focus states.
**Action:** When creating or modifying custom interactive elements (buttons, custom toggles, list items) in `src/webui/template.ts` or similar frontend code, verify they include an explicit `:focus-visible` pseudo-class (e.g., matching the application's established `box-shadow: 0 0 0 3px var(--accent-soft);` pattern) to maintain keyboard accessibility. Take care to combine existing inset box-shadows properly with focus ring box-shadows if both apply.
