## 2025-03-12 - Web UI Template Focus and Loading State Accessbility
**Learning:** In the custom template renderer (`src/webui/template.ts`), accessibility state changes need manual DOM manipulation since there is no reactive UI framework. `aria-busy` needs to be explicitly synced with loading states in utility functions like `busy()`. Additionally, modal dialogs must trap/restore focus properly using explicit `element.focus()` calls (e.g., storing the trigger element and focusing the close button when opened, then returning focus to the trigger element when closed, provided it is still visible).
**Action:** When creating new interactive states or modals in this pure DOM environment, ensure that utility functions (like `busy`) natively manage related ARIA attributes (`aria-busy`), and that explicit `.focus()` logic is included in all modal open/close handlers, accounting for visibility changes of the target elements.
